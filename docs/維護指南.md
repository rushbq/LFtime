# 七號小幫手 · 維護指南

給未來的自己：要調整內容或版面時，先看這份。大部分「資料調整」都不用碰程式邏輯，改設定檔即可。

---

## 1. 專案速覽

- 技術：Vite + React 19 + TypeScript，**純前端、無後端、無資料庫**。
- 沒有 router 套件。首頁 `App.tsx` 用一個 state 切換兩個功能。
- 部署：GitHub Pages。`npm run deploy` = 先 build，再用 gh-pages 把 `dist/` 推到 gh-pages 分支。
- 網址：https://rushbq.github.io/LFtime/ （`vite.config.ts` 的 `base` 是 `/LFtime/`）。

### 檔案地圖
| 檔案 | 作用 |
|---|---|
| `App.tsx` | 外層 shell：右上浮動選單切換工具、回到頂端鈕 |
| `index.css` | 全域設計 tokens（顏色）、浮動選單、回到頂端鈕 |
| `components/ArmsRace/ArmsRace.tsx` | 軍備競賽整個功能（邏輯＋畫面） |

| `components/TimeAssistant.tsx` | 時間助手（三個模式的分頁與外框） |

## 2. 軍備競賽：改資料（`data/armsRaceData.ts`）

整個軍備競賽的排程、分數、建議都在這個檔，改完存檔即可，不用動畫面邏輯。

### 2.1 每日排程 `schedule`
`0`(日)～`6`(六)，每天 6 個時段，值是活動代號（採/建/研/英/戰）：
```ts
schedule: { 1: ["採","建","研","英","戰","採"], ... }
```
代號對應在 `codeToId`（採 → gather…）。時段的時間定義在 `slots`（遊戲時間、對應台灣時間、是否跨隔日 `nextDay`）。

### 2.2 一個活動長怎樣 `activities`
```ts
research: {
  name: "研究科技",
  code: "研",
  color: "#4d8bff",        // 主題色（徽章、時段左邊條、明細側邊會用到）
  boxes: [6000, 15000, 30000], // 三個寶箱門檻（最後一個＝滿箱）
  suggestion: { ... },     // 建議取分（見 2.3）
  gatherPlan: { ... },     // 提前開採推算，目前只有瘋狂採集用（見 2.4）
  groups: [ ... ],         // 明細分組（見 2.5）
}
```

### 2.3 建議取分 `suggestion`
```ts
suggestion: {
  steps: [
    { label: "集結擊殺 8 級變異喪屍", qty: 18, pts: 1600 },
    { label: "集結擊殺 3 級變異喪屍", qty: 2,  pts: 1000 },
  ],
  note: "可留可不留的一段補充說明",
}
```
- 每個 step 有填 `pts` 的話，系統會自動算「合計 ≈ N 分」，達到滿箱門檻會標「（可滿箱）」。
- 不想條列，也可以只放 `note`：`suggestion: { note: "……" }`。
- 側邊顏色會自動用「活動主色調淡 40%」，不會跟時段主色搶視覺（`ArmsRace.tsx` 的 `muted()`）。

### 2.4 提前開採 `gatherPlan`（目前只有瘋狂採集）
```ts
gatherPlan: {
  resources: [
    { name: "鋁土 10 級", hours: 12, detail: "單車採滿時間，非滿箱試算" },
    { name: "鐵", hours: 13, detail: "4 台同採約 25,116 分" },
  ],
  assumption: "試算前提……",
  notes: ["行軍時間提醒……", "礦點容量提醒……"],
}
```
- 系統會用「該場瘋狂採集的台灣開始時間 − hours」自動算出**最晚幾點要開始採集**，跨到前一天會標「前一日」。這是抵達礦點後的實際採集起點，**不含行軍時間**。
- `hours` 可填小數（例如 `12.75` 會顯示為 12 小時 45 分）；`detail` 用來說明該列時間的試算依據。
- 鐵／Z 幣／石頭目前以「最多 4 台車、4 台全採同一資源、目標 25,000 分」試算；混採時以最慢的石頭 14 小時估算。
- `assumption` 顯示共同試算前提；`notes` 以條列顯示行軍、混採與礦點容量等風險。
- 想給別的活動也加這個功能：在那個活動加 `gatherPlan` 即可，畫面會自動出現。

### 2.5 明細分組 `groups`
```ts
groups: [
  {
    title: "集結擊殺 · 變異喪屍 (1–8 級)", // null = 不顯示標題
    chips: true,        // true＝用小方塊網格；不填＝一行行列出
    collapse: true,     // true＝預設收合，給一個「展開 N」按鈕
    summary: "1–35 級：150 → 490，每級 +10", // 收合時顯示這行摘要（取代逐項）
    items: [ ["8 級", 1600], ["7 級", 1500] ], // [名稱, 分數]
  },
]
```
用 `collapse + summary` 就能把很長的等差清單（如喪屍 1–35 級）壓成一行，需要時再展開。

## 3. 時間助手

- 三個模式在 `components/Mode1~3*.tsx`（時間倒數／時間疊加／時間轉換），切換分頁與外框在 `TimeAssistant.tsx`。
- 每個模式有專屬強調色，改 `TimeAssistant.tsx` 的 `TABS` 裡的 `accent` 即可（會套到分頁、區段編號、按鈕、結果框、流程節點）。
- 遊戲比現實慢幾小時：改 `constants.ts` 的 `GAME_TIME_OFFSET_HOURS`。

### 3.1 日期時間輸入 `DateTimeField`
- 共用元件在 `components/shared/DateTimeField.tsx`，把日期與時間拆成兩個原生欄位（`type=date` ＋ `type=time`）。
- **不要改回 `datetime-local`**：桌機 Safari 對它支援不完整，只會顯示日期、無法輸入時間（WebKit 已知限制）。
- 對外的值是 `YYYY-MM-DDTHH:MM` 字串；只填半邊會變成不完整字串，由 `stringToDate` 驗證擋下並顯示錯誤。
- 桌機（可 hover 的裝置）會還原 date/time 欄位的原生外觀——`timeAssistant.css` 裡的 `@media (hover:hover) and (pointer:fine)` 區塊。因為 `appearance:none` 在桌機 Safari 會讓欄位失去內部的可編輯區；手機維持自訂外觀不受影響。

### 3.2 基準時間
- 「時間倒數」「時間疊加」都有可編輯的「基準時間」欄：預設帶入此刻，可改成任何時間點，「更新為現在」按鈕一鍵重設。
- `services/timeUtils.ts` 的 `calculateTimeDifference` / `addDurationToCurrent` 都接受可選的基準時間參數；不傳就用系統時鐘（時間轉換分頁就是這樣用）。

### 3.3 結果窗格的狀態訊號（判斷「我按過計算了嗎」）
- 每次按計算：右上角「於 HH:MM:SS 計算」戳記會更新，且整塊會閃一下（result 元素用 `key={calcCount}` 重掛來重播 CSS 動畫）。
- 任何輸入變動後：舊結果**不清空**，改為變暗（`.result.stale`）並顯示橘色「輸入已變更…」提示（元件內的 `isDirty` state）。

### 3.4 流程軌道 UI
- 倒數＝🕐 基準 → 🚩 目標 → ⏳ 結果；疊加＝🕐 基準 → ＋ 加上 → ＝ 結果。
- 結構：`.flow` > `.flow-step`（grid：左邊 `.fmark` 節點、右邊 `.fbody` 內容），最後一步加 `.act`（按鈕＋結果）。
- 圖示在 `components/shared/FlowIcons.tsx`；連接線／箭頭樣式在 `timeAssistant.css` 的「流程軌道」區塊，節點顏色自動吃分頁 accent。

## 4. 設計系統 / 改顏色

- 全站深色 tokens 在 `index.css` 的 `:root`（`--bg`、`--card`、`--line`、`--good`(綠) …）。
- 兩個功能各自的樣式都有 scope 前綴（`.arms-root` / `.ta-root`），互不干擾。
- 專案**沒有用 Tailwind**（早期的 CDN 已移除），所有樣式都是自己寫的 CSS。

## 6. 指令 / 開發 / 部署

```bash
npm run dev      # 本機開發（http://localhost:5173/LFtime/）
npm run build    # 生產建置到 dist/
npm run deploy   # 建置後推到 GitHub Pages（會自動先 build）
```

### 驗證注意事項（重要）
本專案**沒有裝 `@types/react`**，靠 Vite/esbuild 編譯（只剝型別、不做型別檢查）。
所以 **不要用 `tsc --noEmit` 當關卡**——它會噴一堆假錯（找不到 react、JSX、CSS import）。
要確認有沒有壞，用 **`npm run build`**（能過就是好的）或看 dev server 有沒有報錯。
